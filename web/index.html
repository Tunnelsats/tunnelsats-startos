Lightning Routing Setup Guides

Configure LND to announce the TunnelSats public IP and port, while ensuring normal routing for Tor/clearnet peers.

Append to your lnd.conf: 
# [Application Options]
externalhosts=tunnelsats_ip:tunnelsats_port

# [tor]
tor.skip-proxy-for-clearnet-targets=true
tor.streamisolation=false
Important Note: Restart the LND service on StartOS after modifying its config file to apply the new announced address.

Configure Core Lightning to announce your TunnelSats external IP and port for inbound peer-to-peer connections.

Append to your CLN config file: 
announce-addr=tunnelsats_ip:tunnelsats_port
Important Note: Restart the Core Lightning service on StartOS after saving these modifications.

Frequently Asked Questions

How do I activate my TunnelSats subscription?

You need two things: your WireGuard .conf file and the Enable TunnelSats toggle.

  1. Open the TunnelSats config page in StartOS (⚙ → Configure).
  2. Log in to tunnelsats.com, download your .conf file, and paste its full contents into the WireGuard Configuration field.
  3. Set the Enable TunnelSats toggle to On.
  4. Select your Lightning node (LND or Core Lightning) and click Save.

⚡ Don't forget to also update your lnd.conf or CLN config file — see the Setup Guides section on this dashboard for the exact lines to add.

I saved my config but the tunnel shows "Disconnected" — what do I do?

This is the most common setup issue. Work through these checks in order:

  1. Check the Enable toggle. Go to ⚙ → Configure and confirm Enable TunnelSats is On. Saving the config does not automatically enable the tunnel.
  2. Verify your lnd.conf / CLN config. The tunnel can be connected but your Lightning node won't broadcast correctly until you've added the externalhosts= (LND) or announce-addr= (CLN) lines and restarted the service.
  3. Check the VPN endpoint. The WireGuard handshake needs outbound internet access. If your node is behind a strict firewall, UDP port 51820 must be reachable.
  4. Restart the TunnelSats service. In StartOS, stop and restart TunnelSats after any config change.

Still stuck? Open an issue on GitHub with your dashboard status screenshot.

How do I add the externalhosts / announce-addr lines to my Lightning node persistently?

On StartOS, manual changes to configuration files (like lnd.conf or CLN's config) are normally overwritten on restarts or configuration saves. Use the persistence workarounds below based on your Lightning Node implementation.

Option A: LND Node Configuration

For StartOS 0.3.5.x (Active Version)

Inject the custom host into LND's configuration YAML to prevent the configurator from overwriting it:

  1. SSH into your StartOS host: ssh start9@<your-node-ip>
  2. Open LND's UI configuration: 
    sudo nano /embassy-data/package-data/volumes/lnd/data/main/start9/config.yaml
  3. Locate peer-tor-address, wrap the onion address in quotes, and append a newline (\n) along with your external host setting (replace with your values from Connection Properties above): 
    peer-tor-address: "youronionaddress.onion\nexternalhosts=tunnelsats_ip:tunnelsats_port"
  4. Save (Ctrl+O, Ctrl+X) and restart LND in the StartOS dashboard. The configurator will generate two valid externalhosts entries in LND's configuration.

For StartOS 0.4.x (Upcoming Version)

Configure LND natively via UI actions:

  1. In the StartOS dashboard under the LND service, open the Actions menu.
  2. Select Custom External Host and enter: tunnelsats_ip:tunnelsats_port
  3. Submit and restart LND. This writes to store.json and persistently merges the host on startup.

Option B: Core Lightning (CLN) Node Configuration

For StartOS 0.3.5.x (Active Version)

Inject a config-updating hook into the persistent startup script of the CLN volume:

  1. SSH into your StartOS host: ssh start9@<your-node-ip>
  2. Open the persistent startup script: 
    sudo nano /embassy-data/package-data/volumes/c-lightning/data/main/start9/waitForStart.sh
  3. Add the following block to the end of the file (before the script exits): 
    # Append TunnelSats VPN announce address
VPN_ADDR="tunnelsats_ip:tunnelsats_port"
if ! grep -q "announce-addr=$VPN_ADDR" /root/.lightning/config.main; then
  echo "announce-addr=$VPN_ADDR" >> /root/.lightning/config.main
fi
  4. Save, exit, and restart Core Lightning in the StartOS dashboard. The startup hook will dynamically re-apply your VPN address even if configuration is saved via the GUI.

For StartOS 0.4.x (Upcoming Version)

An enhancement PR has been proposed to the official cln-startos repository to natively introduce a "Custom External Host" UI action identical to LND. Until natively merged, the waitForStart.sh hook can be used to persistently modify CLN's settings.

Is TunnelSats a general VPN for all my StartOS traffic?

No. TunnelSats is purpose-built exclusively for Lightning Network routing.

It tunnels only your Lightning node's inbound peer connections (LND or CLN) — not your Bitcoin node, not your other StartOS apps, and not your general internet traffic. Your Bitcoin node (bitcoind) continues to use Tor as normal.

This is by design: TunnelSats gives your Lightning node a stable, reachable clearnet identity for channel gossip, without exposing your home IP or routing all your traffic through a third party.

If you need a general-purpose VPN for all StartOS system traffic, look for WireGuard or OpenVPN packages in the StartOS app store.

What happens if my subscription expires while the tunnel is active?

This is the most important thing to plan for. When your subscription expires:

  1. The WireGuard tunnel drops — your node can no longer receive inbound connections via the TunnelSats IP.
  2. However, your Lightning node's config files (externalhosts= / announce-addr=) still point to that now-dead IP.
  3. Your node will broadcast an unreachable address to the network, causing peers to fail to connect and potentially affecting channel health.

What to do before expiry:

  • Watch the countdown timer on this dashboard — it turns amber when under 30 days remain.
  • Renew at tunnelsats.com and paste your new config before the old one expires.
  • Or, if you won't renew: disable TunnelSats via ⚙ → Configure (set Enable to Off), remove the externalhosts= / announce-addr= lines from your node config, and restart your Lightning node.
Are my WireGuard config and settings preserved if I uninstall the app?

No — uninstalling TunnelSats from StartOS will delete your configuration data, including your pasted WireGuard config and the enabled/disabled state.

This is a StartOS platform behaviour: when a package is uninstalled, its data volume (/data) is removed along with the container.

Before uninstalling, you should:

  1. Copy your WireGuard .conf content from ⚙ → Configure → WireGuard Configuration and save it somewhere safe.
  2. Disable the TunnelSats tunnel (set Enable to Off and save) so your Lightning node falls back to Tor/clearnet.
  3. Remove the externalhosts= / announce-addr= lines from your Lightning node config and restart it.
  4. Only then uninstall TunnelSats.

⚠ Your subscription itself (purchased at tunnelsats.com) is not affected by uninstalling the StartOS app. If you reinstall, simply paste your .conf file again.
